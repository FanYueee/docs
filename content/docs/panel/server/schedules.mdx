## 建立排程
<Steps>
<Step>
進入排程管理後，點選「**創建排程**」按鈕。
![schedules1](/images/panel/server/schedules/schedules1.png)
</Step>
<Step>
輸入想給排程命名的名稱
![schedules2](/images/panel/server/schedules/schedules2.png)
</Step>
<Step>
以下部分是指排程多久要執行一次，分別有分、時、日、月、星期可以設置，以 Cronjob 格式執行，完成後點擊「**創建排程**」。
![schedules3](/images/panel/server/schedules/schedules3.png)
可以透過 [https://crontab.guru](https://crontab.guru) 來預覽設置您的排程多久會執行。
### CronJob 範例
```
分   時   日 月 星期
*    *    *  *  * -> 表示每分鐘執行
0    0    *  *  * -> 表示每天的 0 時 0 分執行
0    0,12 *  *  * -> 表示每天 0 時 0 分和 12 時 0 分執行
*/10 *    *  *  * -> 表示每 10 分鐘執行一次
0    6-8  *  *  * -> 表示 6~8 時的整點 0 分執行
```
```
*     -> 表示每次(放在分就是每分鐘，放在月就是每個月)
0,6,8 -> 透過逗號(,)隔開，表示**和**的意思
12-18 -> 一槓(-)表示一個區間
*/10  -> 斜線(/)可以表示除法的意思，(如果把這個放在分 */10，表示一小時有 60 分鐘，60/10 表達每 10 分鐘執行一次，一個小時共進行 6 次)
```
</Step>
<Step>
進入該排程後，點擊右上方的「**新排程**」
![schedules4](/images/panel/server/schedules/schedules4.png)
</Step>
<Step>
創建任務有三個欄位
1. 動作
    1. 發送指令（即在控制台發送指令）
    2. 發送電源指令（開機、重啟、關機）
    3. 建立備份
2. 延遲（時間偏移），表示與上一個排程內的任務需要延遲多久才執行。
3. 指令

都完成後點擊創建任務。
![schedules5](/images/panel/server/schedules/schedules5.png)
</Step>
<Step>
完成建立排程
![schedules5](/images/panel/server/schedules/schedules5.png)
</Step>
</Steps>
